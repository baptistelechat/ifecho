# BDR-070 — J+2 filter : toutes les 24h, pas `< nowHour`

| Décision                                                        | Pourquoi                                                                                                                                                        | Alternatives considérées                                                                    | Statut |
| --------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------- | ------ |
| Filtre J+2 = `date === dayAfterDate` sans restriction `hourNum` | Le filtre `< nowHour` créait une borne glissante : à 17h04, J+2 s'arrêtait à l'heure 16, fin de créneau = "17h". Chaque heure écoulée effaçait une heure de J+2 | Garder `< nowHour` pour borner strictement à ~48h — refusé car c'est la cause racine du bug | actif  |
