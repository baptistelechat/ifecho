# BDR-055 — `controllerchange` + auto-reload silencieux pour MAJ PWA

| Décision                                                                                                                                                 | Pourquoi                                                                                                                                                                                                 | Alternatives considérées                                                                                                                        | Statut |
| -------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------- | ------ |
| `navigator.serviceWorker.addEventListener('controllerchange', ...)` + `window.location.reload()` avec guard `hadController` dans `useUpdateNotification` | Fiable sur tous les cycles de mise à jour, zéro friction utilisateur, compatible `registerType: "autoUpdate"`. Sur une SPA stateless les données sont fraîches à chaque load — le reload est transparent | `useRegisterSW` + Sonner toast Sonner (abandonné — `onNeedRefresh` non fiable après 2e cycle, voir [ZBLK-033](../archive/blockers/ZBLK-033.md)) | actif  |
