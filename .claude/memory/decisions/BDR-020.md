# BDR-020 — `deltaT` = ressenti intérieur − ressenti extérieur

| Décision                                                                                                                                                    | Pourquoi                                                                                                                                                                                                          | Alternatives considérées                                                                                                               | Statut |
| ----------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------- | ------ |
| `deltaT = (indoorTemp + comfortBias) - apparentTemperature` dans `useVentilationScore` — `comfortBias` intégré dans le delta, score = `deltaT + bonusNight` | `deltaT` doit refléter l'écart ressenti réel (les deux côtés en "ressenti"). Les consommateurs (`ThermalDelta`, `VerdictBanner`) lisent `currentScore.deltaT` directement sans recalculer avec `indoorTemp` brut. | Garder `comfortBias` dans le score (existant) — mathématiquement identique mais `deltaT` sémantiquement incorrect (gap brut non perçu) | actif  |
