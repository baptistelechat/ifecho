# BDR-054 — Layout headers PostHog : text tiles w=12 + reorder preserve

| Décision                                                                                                                                                                                                                                      | Pourquoi                                                                                                                                                                                                                                    | Alternatives considérées                                                                          | Statut |
| --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------- | ------ |
| Tiles texte de section = `w=12, h=2` (pleine largeur) ; insights = `w=6` (2 colonnes). Workflow : `dashboard-update-text-tile` avec `layouts: {sm: {w: 12, h: 2}}` sur chaque header, puis `dashboard-reorder-tiles` avec `layout: preserve`. | Les headers occupant une ligne entière créent une séparation visuelle claire entre les 6 sections. Avec `w=6` (résultat d'un `two_column`), les headers partageaient la ligne avec le premier insight de leur section — confusion visuelle. | All tiles `two_column` : headers partagent la ligne avec un insight, pas de séparation de section | actif  |
