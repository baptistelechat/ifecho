# BDR-065 — Dashboard UTM restructuré en 2 sections (Vue globale + Partage viral)

| Décision                                                                                                                                | Pourquoi                                                                                                                         | Alternatives considérées                                                                  | Statut |
| --------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------- | ------ |
| 2 sections : "📊 Vue globale UTM" (toutes sources, filtre `utm_source is_set`) + "🔁 Partage viral (share)" (filtres spécifiques share) | Les 5 insights originaux filtraient `utm_source=share` uniquement — la campagne "launch" n'apparaissait jamais dans le dashboard | Section unique avec breakdown `utm_campaign` seulement / garder les 5 insights tels quels | actif  |

## Contexte

Dashboard "Acquisition — Liens UTM" (ID 769548). Avant : 5 insights spécifiques au partage viral (bouton ShareButton). Après : section Vue globale (BoldNumber total + LineGraph tendance + Pie par `utm_campaign`) + section Partage viral (3 insights share-specific réorganisés).

Date range corrigé au passage : `explicitDate: true` figé sur `2026-06-25T10:37:25` → `explicitDate: false` + `date_from: "-30d"` sur les 5 insights existants et le dashboard.
