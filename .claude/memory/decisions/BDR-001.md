# BDR-001 — Vite + React (pas Next.js) pour SPA statique V0

| Décision                      | Pourquoi                                                                       | Alternatives considérées             | Statut |
| ----------------------------- | ------------------------------------------------------------------------------ | ------------------------------------ | ------ |
| Vite + React + TypeScript SPA | App 100% client-side, pas de SSR nécessaire, setup plus rapide en contexte 36h | Next.js (trop lourd), CRA (déprécié) | actif  |

L'app ifecho est entièrement client-side : géolocalisation browser, appel Open-Meteo directement depuis le client, génération `.ics` côté client. Aucune donnée ne nécessite un rendu serveur. Vite permet un démarrage plus rapide et un build statique déployable sur Vercel en quelques minutes.
