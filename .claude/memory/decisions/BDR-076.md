# BDR-076 — `forecast_days: "2"` + filtre date J/J+1 strict

| Décision                                                                                      | Pourquoi                                                                                                                                                                                                  | Alternatives considérées                                                                                          | Statut |
| --------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------- | ------ |
| `forecast_days: "2"` + filtre par date string `date === todayDate \|\| date === tomorrowDate` | `forecast_days: "3"` = 72h de données. Fenêtre ms de 28h depuis samedi soir peut atteindre lundi 0h → heures lundi affichées un samedi. Filtre date string : immunisé contre les décalages ms inter-jours | Garder `forecast_days: "3"` + filtre strict (données inutiles), fenêtre ms réduite à 24h (risque de rater minuit) | actif  |
