# BDR-002 — Rappel `.ics` côté client — pas de push VAPID V0

| Décision                        | Pourquoi                                                                            | Alternatives considérées                                | Statut |
| ------------------------------- | ----------------------------------------------------------------------------------- | ------------------------------------------------------- | ------ |
| Fichier `.ics` généré en client | Zero infra, zero compte, 100% compatibilité — VAPID nécessite Edge Function + infra | FCM Topics + RPi (V1), notifs web VAPID (trop complexe) | actif  |

Le rappel calendrier n'est pas "moins bien" qu'une notif push — c'est juste différent. 100% des utilisateurs ont un calendrier iOS/Google. Un tap → le téléphone rappelle à l'heure cible. Génération du `.ics` entièrement côté client avec un array de chaînes jointées.

La V1 prévoira FCM Topics avec un cron sur RPi (coût fixe O(départements), scalable à l'infini).
