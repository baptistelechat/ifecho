# LRN-030 — Afficher la valeur sur laquelle repose le seuil, pas une valeur dérivée

| Pattern observé                                                                                                                                                             | Contexte                                                                                                                                                             | Application future                                                                                                                   |
| --------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------ |
| Quand couleur/verdict repose sur `score` mais le message affiche `deltaT`, l'utilisateur voit "1.1°C" avec une bannière verte sans comprendre pourquoi — confusion garantie | Bug ifecho : VerdictBanner utilisait `deltaT` (1.1°C → jaune) pendant que ThermalDelta et Timeline utilisaient `score` (2.1 → vert) grâce au `bonusNight` silencieux | Toujours afficher la valeur sur laquelle repose la décision de seuil, ou expliquer explicitement le bonus/malus qui crée le décalage |
