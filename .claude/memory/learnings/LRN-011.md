# LRN-011 — Affordance "thermostat" != stepper +/- : l'argument juste = precision sans ciblage

| Pattern observe                                                                                                                                                                                                                      | Contexte                                                                                                                                                       | Application future                                                                                                                                                           |
| ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Justifier un stepper +/- par "thermostat" est faux : les thermostats physiques sont des cadrans rotatifs (Nest = anneau), pas des boutons. Argument correct : pas de ciblage de precision requis, 1 tap = 1 unite, bornes implicites | Debat Rodin sur slider vs stepper pour la temperature interieure dans ifecho. Metaphore thermostat ecartee car elle designe un cadran, pas des boutons up/down | Pour toute decision UI justifiee par une metaphore physique : tester la metaphore strictement ("est-ce bien CETTE interaction que l'objet implemente ?") avant de l'invoquer |
