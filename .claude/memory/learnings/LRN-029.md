# LRN-029 — `.filter().map()` / `.map().filter()` → `.flatMap()` single-pass

| Pattern observé                                                                                                                                                                                                                                                                       | Contexte                                                                                                                    | Application future                                                                                                           |
| ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------- |
| react-doctor (règle `js-combine-iterations`) détecte les chaînes `.filter().map()` ou `.map().filter()` comme double-pass évitable. Fix idiomatique : `items.flatMap(item => { if (!keep(item)) return []; return [transform(item)]; })` — single-pass, lisible, compatible TS strict | `IdealSlots.tsx` et `useVentilationScore.ts` utilisaient `.filter().map()` et `.map().filter()` — convertis en `.flatMap()` | Préférer `.flatMap()` dès qu'on filtre ET transforme en une seule opération — plus efficace et élimine la règle react-doctor |
