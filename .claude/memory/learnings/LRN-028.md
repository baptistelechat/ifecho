# LRN-028 — `pnpm-workspace.yaml` hardening : champs racine

| Pattern observé                                                                                                                                                                                                                                                                        | Contexte                                                                                                       | Application future                                                                                             |
| -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------- |
| react-doctor attend `minimumReleaseAge` et `trustPolicy` directement à la racine de `pnpm-workspace.yaml`, **pas** sous une clé `settings:`. Avec `settings: { minimumReleaseAge: 10080 }`, le fichier existe mais les diagnostics `require-pnpm-hardening` persistent silencieusement | Création initiale avec structure `settings:` — 2 warnings persistaient alors que le fichier était bien présent | Toujours écrire `minimumReleaseAge: 10080` et `trustPolicy: no-downgrade` à la racine du fichier, sans wrapper |
