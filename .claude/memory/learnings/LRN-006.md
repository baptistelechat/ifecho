# LRN-006 — Variable API composite : ne pas doubler ses composantes

| Pattern observé                                                                                                                                                                 | Contexte                                                                                                   | Application future                                                                                             |
| ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------- |
| `apparent_temperature` Open-Meteo intègre temp sèche + vent + humidité + rayonnement. Ajouter des malus/bonus manuels pour ces mêmes facteurs crée un double comptage invisible | Algo ventilation ifecho — `malusHumidity` (−3 si humidité >80%) supprimé car déjà capturé dans le ressenti | Avant d'ajouter un modificateur manuel basé sur X, vérifier si la variable principale composite intègre déjà X |
