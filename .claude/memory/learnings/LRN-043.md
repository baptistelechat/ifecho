# LRN-043 — `preconnect` sans `crossorigin` = inefficace pour ressources CORS

| Pattern observé                                                                                                                                                                                                           | Contexte                                                                                                             | Application future                                                                                   |
| ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------- |
| `<link rel="preconnect">` sans `crossorigin` est silencieusement ignoré pour les ressources chargées via `fetch()` (CORS). Le navigateur ouvre deux connexions au lieu d'une. Lighthouse signale l'optimisation manquante | `index.html` — `preconnect` vers `api.open-meteo.com` et `geocoding-api.open-meteo.com` complétés avec `crossorigin` | Toujours ajouter `crossorigin` sur les preconnect hints pointant vers des API appelées via `fetch()` |
