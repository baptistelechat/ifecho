# LRN-005 — `prevProp` inline render : fix `no-adjust-state-on-prop-change`

| Pattern observé                                                                                                                                                             | Contexte                                                                                                                                                | Application future                                                                                                                                                                     |
| --------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Quand `setState` (loading/error) est en tête d'un `useEffect([prop])`, le déplacer dans un bloc `if (prop !== prevProp)` inline pendant le render avec `useState(prevProp)` | `useWeatherForecast` : `setIsLoading(true)` + `setError(null)` dans `useEffect([lat, lon])` → flaggés `no-adjust-state-on-prop-change` par React Doctor | Toute fois que React Doctor ou les React docs signalent ce pattern : créer `useState(prevProp)`, bloc `if` inline, l'effect ne contient plus que l'async et la mise à jour du résultat |
