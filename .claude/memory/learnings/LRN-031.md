# LRN-031 — `apparent_temperature` Open-Meteo exclut le rayonnement solaire

| Pattern observé                                                                                                                                                                                                | Contexte                                                                                                                                                        | Application future                                                                                                 |
| -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------ |
| `apparent_temperature` (Open-Meteo) intègre wind chill + humidity index mais PAS l'irradiance solaire — pour modéliser l'effet du soleil, utiliser `uvIndex` (champ distinct disponible dans le même endpoint) | ifecho — remise en question du `bonusNight` : le rayonnement solaire n'était pas dans `apparent_temperature`, d'où le besoin d'un champ séparé pour `uvPenalty` | Tout projet météo Open-Meteo où le gain solaire par les vitres ou l'exposition au soleil est un facteur de confort |

- [BDR-029](../decisions/BDR-029.md) — `uvPenalty` remplace `bonusNight` dans le score ventilation
