# LRN-041 — WCAG 2.5.3 : `aria-label` doit contenir le texte visible comme substring

| Pattern observé                                                                                                                                                                                                                                                                   | Contexte                                                                      | Application future                                                                                                                                                                           |
| --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Si un bouton a un texte visible ("Me localiser") et un `aria-label` différent ("Utiliser ma position GPS"), Lighthouse signale `label-content-name-mismatch` (WCAG 2.5.3). Le nom accessible doit contenir le texte visible — ou supprimer l'`aria-label` si le texte suffit seul | Bouton "Me localiser" dans `LocationSearch/index.tsx` — `aria-label` supprimé | Avant d'ajouter un `aria-label` sur un élément avec texte visible : vérifier que l'attribut inclut le texte visible comme substring, ou simplement supprimer si le texte est déjà descriptif |
