# LRN-004 — FCM Topics : coût O(topics), pas O(users)

| Pattern observé                                                                                                                                                                              | Contexte                                                                                                                                | Application future                                                                                                                                                  |
| -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Avec FCM Topics, 1 appel API = N utilisateurs notifiés simultanément. Le coût d'infrastructure (fetches météo + appels FCM) est O(nombre de topics), indépendant de la taille de l'audience. | Brainstorm ifecho — le modèle VAPID individuel semblait nécessiter un fetch par user. FCM Topics change complètement le modèle de coût. | Pour tout système de push multi-utilisateurs : regrouper les users par dimension fixe (région, catégorie, préférence) → coût constant quelle que soit la croissance |
