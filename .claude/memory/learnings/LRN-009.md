# LRN-009 — État React `T | null` avec défaut évident → non-nullable `T`

| Pattern observé                                                                                     | Contexte     | Application future |
| --------------------------------------------------------------------------------------------------- | ------------ | ------------------ | --------------------------------------------------------------------------------------------------------------------------- | ----------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Si un état a une valeur "toujours valide" par défaut, préférer `useState<T>(default)` à `useState<T | null>(null)` | `ComfortLevel      | null`migré vers`ComfortLevel`avec`"neutral"`. Clé : "y a-t-il un état métier 'rien de sélectionné' ?" — si non, pas de null | Avant d'ajouter ` | null` à un state, poser la question. Si le défaut est évident → non-nullable, simplifie toute la chaîne de types et supprime les ternaires défensifs partout |
