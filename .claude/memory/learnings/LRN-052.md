# LRN-052 — Fichiers d'overlap lors de refacto parallèle → réserver au coordonnateur

| Pattern observé                                                                                                                                                                                         | Contexte                                                                                                                                                                                                                                                | Application future                                                                                                                                   |
| ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------- |
| Quand une refacto touche N fichiers selon plusieurs axes distincts, certains fichiers sont inévitablement touchés par 2 axes — les réserver au coordonnateur plutôt que de les déléguer à un sous-agent | 7 findings ponytail sur 2 axes (extraction utils + analytics singleton). App.tsx, VentilationTimeline et TipsSection auraient eu 2 agents simultanés. Fix : agents sur sets strictement disjoints, coordonnateur traite les overlaps manuellement après | Identifier les overlaps avant tout fan-out. Si un fichier apparaît dans 2 buckets d'agents → l'extraire et le traiter manuellement en fin de session |
