# LRN-017 — Continuité de créneaux : diff timestamps plutôt que `hour === last + 1`

| Pattern observé                                                                                                                                                                 | Contexte                                                                              | Application future                                                                                  |
| ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------- |
| `hour === last + 1` casse silencieusement à la frontière 23→0 (23 + 1 = 24 ≠ 0). Fix : diff de timestamps ≤ 3_601_000 ms (1h + 1s de tolérance) entre deux entrées consécutives | `getIdealSlots()` dans `IdealSlots.tsx` — créneaux favorables qui traversaient minuit | Tout algorithme de groupement de créneaux horaires : comparer les timestamps, pas les heures brutes |
