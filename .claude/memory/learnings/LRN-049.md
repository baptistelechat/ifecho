# LRN-049 — PostHog MCP `dashboard-tile-copy` : réponse oversized ≠ échec opérationnel

| Pattern observé                                                                                                                                                                                                                                                                                                                        | Contexte                                                                                                                                          | Application future                                                                                                                                                                                                                                                   |
| -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Chaque `dashboard-tile-copy` retourne l'état complet du dashboard cible. Le payload grossit à chaque ajout : passé ~15–20 tiles, la réponse dépasse le quota tokens MCP et génère "result exceeds maximum allowed tokens". L'opération a quand même réussi — confirmer via `_posthogUrl` dans le preview ou un `dashboard-get` séparé. | Cleanup dashboard ifecho V2 (2026-06-23) : 8 copies depuis V2-temp → V2-master, toutes "oversized" en réponse, toutes opérationnelles en réalité. | Ne pas relancer une copy pensant qu'elle a échoué. Après erreur oversized sur un MCP PostHog write, vérifier l'état réel via un read (`dashboard-get`) plutôt que supposer l'échec. Ce pattern s'applique à tout write MCP dont la réponse est la ressource entière. |
