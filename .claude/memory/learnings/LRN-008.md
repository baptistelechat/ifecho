# LRN-008 — Structural typing event handler : évite l'import namespace React

| Pattern observé | Contexte | Application future |
| --- | --- | --- |
| `React.KeyboardEvent<HTMLInputElement>` dans un projet n'important que les named imports (pas le namespace `React`) → TS error → Biome hook revert du fichier entier. Fix : structural typing `(e: { key: string })` — duck-typed par TypeScript, 0 import supplémentaire. | `handleKeyDown` pour Enter/Escape dans l'édition inline. Premier Write déclenche le revert automatique de Biome. | Préférer le structural typing pour les event handlers simples : `(e: { key: string })`, `(e: { value: string })`, etc. Remplace `React.KeyboardEvent`, `React.ChangeEvent` sans polluer les imports. |
