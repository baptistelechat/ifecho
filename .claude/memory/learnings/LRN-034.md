# LRN-034 — `AnimatePresence mode="wait"` → flash lors transition état-vide↔contenu

| Pattern observé                                                                                                                                                                                                                       | Contexte                                                                                                                                     | Application future                                                                                                                                             |
| ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `AnimatePresence mode="wait"` sur un parent contenant un état vide + des items : pendant la transition, le parent reste visible (header, bordures, padding) mais les enfants sont absents → flash d'une card vide entre exit et enter | IdealSlots — transition entre `slots.length === 0` et présence de slots ; le header "Créneaux idéaux à venir" restait visible pendant le gap | Ne jamais utiliser `mode="wait"` si le composant a un état structurel persistant (header, bordures, padding) — préférer une structure plate sans `mode="wait"` |
