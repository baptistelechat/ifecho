# LRN-038 — Vite HMR ENOENT après rename fichier → clear `node_modules/.vite` + restart

| Pattern observé                                                                                                                                                         | Contexte                                                                                                                                                             | Application future                                                                                                                                                   |
| ----------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Renommer un fichier source pendant une session Vite active cause `ENOENT: no such file or directory` au prochain accès — le module graph garde l'ancien chemin en cache | `favicon.svg` → `logo.svg` pendant dev server actif ; l'erreur apparaît dans `vite:css-analysis` même si aucune référence au fichier ne subsiste dans le code source | Fix : supprimer `node_modules/.vite/` + redémarrer le dev server ; ne pas chercher de référence orpheline dans le code — c'est un problème de cache HMR, pas de code |
