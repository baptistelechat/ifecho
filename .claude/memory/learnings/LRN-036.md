# LRN-036 — `min-h` stabilise la hauteur au plancher du cas long

| Pattern observé                                                                                                                                                                                                            | Contexte                                                                                                                                                                                                                                                            | Application future                                                                                                                                                                                                                           |
| -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `min-h` empêche le rétrécissement d'un composant à contenu variable en fixant la hauteur minimale à celle du **cas le plus long** — pas le plus court. Le court est "tiré vers le haut", le long reste libre de s'étendre. | VerdictBanner : message 1 ligne ("Bon moment pour aérer") vs 2 lignes ("Ne pas aérer") → en passant du long au court, la card rétrécissait et décalait les boutons stepper sous le curseur lors de clics rapides. Fix : `min-h-[6.5rem]` = hauteur du cas 2 lignes. | Tout composant avec contenu variable (verdicts, labels, descriptions) dont le changement de hauteur déplacerait des éléments interactifs adjacents (boutons, sliders). Valeur à estimer empiriquement puis vérifier visuellement sur mobile. |
