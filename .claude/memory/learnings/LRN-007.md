# LRN-007 — `<input>` intrinsic height > `<button>` : fix conteneur `h-[1em]` + `h-full`

| Pattern observé | Contexte | Application future |
| --- | --- | --- |
| Remplacer un `<button>` par un `<input>` inline cause un layout shift même avec `p-0 border-0` car `<input>` a une intrinsic height plus grande définie par le navigateur, pas par padding/border. Fix : envelopper dans `div.flex` à `h-[1em]` (= hauteur du texte courant) et passer `h-full` à l'input. | Édition inline de la température intérieure — toggle lecture/édition sur le chiffre `26°`. Premier fix `p-0 border-0` inefficace, confirmé par screenshots dev-browser before/after. | Tout toggle lecture/édition inline en React : poser `h-[1em]` sur le conteneur wrapper, `h-full` sur l'input → hauteur partagée exacte, 0 layout shift. |
