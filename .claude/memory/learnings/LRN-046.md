# LRN-046 — Hash routing SPA : `useState` init + `hashchange` + `history.replaceState`

| Pattern observé                                                                                                                                                                                                                                                                        | Contexte                                                                                 | Application future                                                                                |
| -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------- |
| Pour une SPA avec une seule route secondaire, le routing hash natif suffit : `useState(() => window.location.hash === '#page')` pour l'état initial, `addEventListener('hashchange')` pour les changements, `history.replaceState(null, '', pathname)` pour nettoyer le hash au retour | ifecho SPA Vite + React, route `#confidentialite` vers PrivacyPage sans react-router-dom | Toute SPA légère n'ayant qu'une ou deux routes secondaires, où react-router serait surdimensionné |
