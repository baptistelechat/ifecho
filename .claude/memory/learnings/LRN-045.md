# LRN-045 — Event analytics composé → consumer multi-hooks, pas hook fetch

| Pattern observé                                                                     | Contexte                                                                  | Application future                                            |
| ----------------------------------------------------------------------------------- | ------------------------------------------------------------------------- | ------------------------------------------------------------- |
| `weather_loaded` placé dans App.tsx car nécessite `bestHour` calculé par un 2e hook | Event nécessitant données de `useWeatherForecast` + `useVentilationScore` | Tout event analytics nécessitant données de sources multiples |

## Règle

L'event analytics se place là où **toutes ses données sont disponibles**, pas là où la donnée "primaire" est fetchée.

```ts
// ✅ Dans App.tsx — accès à weather (useWeatherForecast) ET bestHour (useVentilationScore)
useEffect(() => {
  if (!weather || !location) return;
  analytics.weatherLoaded({
    city: location.city,
    bestHour: bestHour?.hour ?? null,
    topScore: bestHour?.score ?? 0,
  });
  // eslint-disable-next-line react-hooks/exhaustive-deps
}, [weather]);

// ❌ Dans useWeatherForecast — bestHour indisponible à ce niveau
```
