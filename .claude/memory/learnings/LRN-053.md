# LRN-053 — Guard `startMs >= now + WINDOW_MS` dans `getIdealSlots`

| Pattern observé                                                                                                                                                                                                | Contexte                                                                                                         | Application future                                                                                                                            |
| -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------- |
| Quand les données s'étendent à 48h mais l'affichage reste à 24h, filtrer les groupes de créneaux par `startMs < now + WINDOW_MS` évite d'afficher des créneaux qui "démarrent" dans la zone de données étendue | `getIdealSlots` reçoit 48h de scores ; sans guard, un créneau démarrant à now+26h serait proposé à l'utilisateur | Tout algorithme de grouping de créneaux ou d'events avec fenêtre de données plus large que la fenêtre d'affichage — calendrier, scheduling UI |
