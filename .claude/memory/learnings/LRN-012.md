# LRN-012 — Biais additif intégré dans le delta sémantique, pas le score

| Pattern observé                                                                                                                                                                                                | Contexte                                                                                                                                                                                                                                           | Application future                                                                                                                                                                       |
| -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Quand un algo de score additionne delta + biais de même unité, intégrer le biais dans le delta préserve la sémantique du champ stocké et supprime la responsabilité de reconstruction chez chaque consommateur | `useVentilationScore` : `deltaT` était brut (`indoorTemp - apparentTemperature`) + `comfortBias` séparé dans `score`. `ThermalDelta` et `VerdictBanner` devaient recalculer localement avec `indoorTemp` brut plutôt que lire `deltaT` directement | Si un champ `deltaX` est stocké dans un type partagé, s'assurer qu'il encode la totalité de l'écart sémantique avant stockage — les consommateurs ne doivent pas avoir à le reconstituer |
