# LRN-048 — PostHog `insight-update` exige la query complète (pas de merge partiel)

| Pattern observé                                                                                                                                                                                                                                        | Contexte                                                                                        | Application future                                                                                                                  |
| ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ----------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------- |
| L'endpoint `insight-update` remplace la query entière — pas de patch partiel. Pour modifier un flag (`filterTestAccounts: true`), il faut : (1) fetch l'insight, (2) modifier le flag dans la structure en mémoire, (3) re-soumettre le query complet. | Mise à jour de 17 insights en 3 batches : fetch parallèle → patch en mémoire → update parallèle | Toujours récupérer l'insight avant de le modifier. Sur un grand nombre d'insights, batcher les fetches et les updates en parallèle. |
