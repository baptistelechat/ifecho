# LRN-024 — `touch-action: pan-y` requis pour swipe horizontal sur mobile

| Pattern observé                                                                                                                                                                                | Contexte                                                                                    | Application future                                                                                                                     |
| ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------- |
| Sans `touch-action: pan-y`, le navigateur mobile intercepte tous les gestes de touch pour son scroll natif — `onPointerDown`/`onPointerUp` ne se déclenchent jamais lors d'un swipe horizontal | Carousel TipsSection : swipe inopérant sur mobile malgré pointer events correctement câblés | Appliquer `[touch-action:pan-y]` (syntaxe arbitrary property Tailwind v4) sur tout container swipeable avant même de tester sur device |
