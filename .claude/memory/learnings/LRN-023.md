# LRN-023 — AnimatePresence mode="wait" + variants : card initiale invisible

| Pattern observé                                                                                                                                                                                                                              | Contexte                                                                                              | Application future                                                                                                                                                                          |
| -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `mode="wait"` avec `custom` variants applique l'état "exit" sur la card montée initialement — elle reste à `opacity: 0, translateX(100%)` sans jamais entrer dans sa phase `initial → animate`. `initial={false}` ne résout pas le problème. | Carousel TipsSection : première card invisible après mount, impossible à corriger via AnimatePresence | Pour tout carousel React avec framer-motion, utiliser le pattern CSS flex-translate : `motion.div animate={{ x: '-{N*100}%' }}` sur un flex strip `min-w-full` — plus fiable et plus simple |
