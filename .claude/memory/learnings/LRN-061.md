# LRN-061 — Fallback `getHighestColor` dans `getActiveColor` : icône colorée après expiration

| Pattern observé                                                                                                              | Contexte                                                                                                                        | Application future                                                                                                                                                      |
| ---------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Quand toutes les plages J sont expirées et qu'aucune future n'existe, retourner `getHighestColor(today)` plutôt que `"vert"` | Après 16h, l'icône orages devenait verte malgré une alerte jaune qui avait existé ce jour — dégradant la perception utilisateur | Dans tout `getActive*` avec fallback couleur : distinguer "aucune alerte ce jour" (→ vert) vs "alerte expirée ce jour" (→ couleur max historique via `getHighestColor`) |
